# Подключение базы знаний к AI-ассистенту

Этот документ описывает, как подключить базу знаний к AI-ассистенту через OpenAI Assistants API с Vector Store.

## Что такое Vector Store

Vector Store — это хранилище документов в OpenAI, которое позволяет ассистенту искать информацию в загруженных файлах и отвечать на основе этих данных.

## Шаги подключения

### 1. Подготовка файлов базы знаний

Соберите все документы, которые должен знать ассистент:
- FAQ
- Описания продуктов
- Инструкции
- Прайс-листы
- Любые другие текстовые материалы

Поддерживаемые форматы:
- `.txt`
- `.pdf`
- `.docx`
- `.md`

### 2. Создание Vector Store в OpenAI

1. Перейдите в [OpenAI Platform](https://platform.openai.com/)
2. Откройте раздел **Storage** → **Vector Stores**
3. Нажмите **Create Vector Store**
4. Загрузите файлы базы знаний
5. Скопируйте **Vector Store ID** (формат: `vs_...`)

### 3. Создание Assistant

1. Откройте раздел **Assistants**
2. Нажмите **Create Assistant**
3. Настройте:
   - **Name**: название ассистента
   - **Instructions**: системный промпт (см. пример ниже)
   - **Model**: рекомендуется `gpt-4o-mini` или `gpt-4o`
   - **Tools**: включите **File Search**
   - **Vector Store**: выберите созданный Vector Store
4. Скопируйте **Assistant ID** (формат: `asst_...`)

### 4. Настройка backend

Добавьте в `backend/.env`:

```env
OPENAI_API_KEY=sk-...
OPENAI_ASSISTANT_ID=asst_...
VECTOR_STORE_ID=vs_...
```

**Важно**: Если `OPENAI_ASSISTANT_ID` указан, backend автоматически использует Assistants API с базой знаний. Если не указан — работает стандартный Chat Completions API без базы знаний.

### 5. Перезапуск сервиса

```bash
docker compose restart chat-backend
```

Или если запускаете локально:

```bash
cd backend
python main.py
```

## Пример системного промпта для Assistant

```
Ты — AI-ассистент первой линии поддержки по ростовым куклам и комплектующим.

Твоя задача:
- Отвечать на вопросы клиентов о продукции на основе базы знаний
- Помогать с выбором кукол и комплектующих
- Предоставлять точную информацию о характеристиках и ценах
- Быть вежливым, профессиональным и кратким

Правила:
- Используй только информацию из базы знаний
- Если информации нет в базе, честно скажи об этом
- Не придумывай цены или характеристики
- При сложных вопросах предлагай связаться с менеджером: https://t.me/eth1r

Стиль общения:
- Дружелюбный, но профессиональный
- Краткие и понятные ответы
- Структурированная подача информации
```

## Как работает интеграция

Backend автоматически определяет режим работы:

1. **С базой знаний** (если `OPENAI_ASSISTANT_ID` указан):
   - Создается отдельный thread для каждой сессии
   - Сообщения добавляются в thread
   - Assistant обрабатывает запрос с доступом к Vector Store
   - Ответ возвращается из thread

2. **Без базы знаний** (если `OPENAI_ASSISTANT_ID` не указан):
   - Используется стандартный Chat Completions API
   - История хранится в памяти backend
   - Работает быстрее, но без доступа к документам

## Проверка работы

После настройки проверьте:

1. Откройте чат-виджет на сайте
2. Задайте вопрос из базы знаний
3. Проверьте логи backend:
   ```bash
   docker compose logs -f chat-backend
   ```
4. В логах должно быть: `Using Assistants API with assistant_id: asst_...`
5. Убедитесь, что ассистент отвечает на основе загруженных документов

## Обновление базы знаний

Чтобы обновить базу знаний:

1. Откройте Vector Store в OpenAI Platform
2. Загрузите новые файлы или удалите старые
3. Перезапуск backend не требуется — изменения применяются автоматически

## Troubleshooting

### Ассистент не использует базу знаний

- Проверьте логи: должно быть `Using Assistants API with assistant_id`
- Проверьте, что File Search включен в настройках Assistant
- Убедитесь, что Vector Store привязан к Assistant
- Проверьте формат загруженных файлов

### Медленные ответы

- Assistants API может работать медленнее обычного Chat API (10-30 секунд)
- Увеличьте `OPENAI_TIMEOUT` в `.env` (по умолчанию 30 секунд)
- Рассмотрите использование более быстрой модели

### Ошибки в логах

Проверьте логи:
```bash
docker compose logs -f chat-backend
```

Типичные ошибки:
- `Assistant timeout after Xs` — увеличьте OPENAI_TIMEOUT
- `Assistant run failed` — проверьте настройки Assistant в OpenAI Platform
- `Assistant run was cancelled` — попробуйте снова, возможно временная проблема
- `Rate limit` — проверьте лимиты вашего OpenAI аккаунта

### Переключение между режимами

Чтобы переключиться:
- **На Assistants API**: добавьте `OPENAI_ASSISTANT_ID` в `.env`
- **На Chat Completions**: удалите или закомментируйте `OPENAI_ASSISTANT_ID`
- Перезапустите backend
