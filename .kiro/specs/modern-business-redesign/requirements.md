# Requirements Document

## Introduction

Глобальный редизайн портфолио-сайта с целью трансформации визуального стиля от "терминала разработчика" к современному, визуально дорогому и бизнес-ориентированному дизайну. Редизайн должен сохранить функциональность, но кардинально изменить визуальное восприятие для привлечения бизнес-заказчиков.

## Glossary

- **System**: Портфолио-сайт (React-приложение)
- **ProjectCard**: Компонент карточки проекта
- **Glassmorphism**: Стиль дизайна с полупрозрачным фоном и эффектом размытия
- **CTA**: Call-to-Action элементы (кнопки призыва к действию)
- **Metrics**: Показатели качества проектов (Faithfulness, Precision и т.д.)
- **Code_Elements**: Элементы кода (<code>, <pre>, теги в Лаборатории)

## Requirements

### Requirement 1: Современная Типографика

**User Story:** Как бизнес-заказчик, я хочу видеть читабельный современный текст, чтобы сайт выглядел профессионально и не напоминал терминал.

#### Acceptance Criteria

1. THE System SHALL использовать современный sans-serif шрифт ('Inter', system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif) для всех основных текстовых элементов (body, h1-h6, p)
2. THE System SHALL сохранять monospace шрифт ТОЛЬКО для Code_Elements (<code>, <pre>, теги в Лаборатории, содержимое промптов)
3. THE System SHALL использовать межстрочный интервал (line-height) не менее 1.6 для основного текста
4. WHEN пользователь просматривает любую страницу, THEN все заголовки и параграфы SHALL отображаться с новым шрифтом
5. WHEN пользователь просматривает Лабораторию, THEN элементы кода SHALL сохранять monospace шрифт

### Requirement 2: Премиальная Цветовая Палитра

**User Story:** Как бизнес-заказчик, я хочу видеть глубокую, богатую цветовую схему, чтобы сайт выглядел дорого и современно.

#### Acceptance Criteria

1. THE System SHALL использовать глубокий темный фон (#0F1117 или #0a0a0a) вместо чисто черного (#000000)
2. THE System SHALL применять акцентный градиент (linear-gradient(135deg, #3A86FF 0%, #8338EC 100%)) для CTA элементов
3. THE System SHALL применять акцентный градиент для ключевых заголовков
4. WHEN пользователь загружает сайт, THEN фон SHALL отображаться в глубоком темном оттенке
5. WHEN пользователь наводит на кнопки, THEN градиент SHALL быть визуально заметен

### Requirement 3: Glassmorphism Эффект для Карточек

**User Story:** Как бизнес-заказчик, я хочу видеть современные стеклянные карточки проектов, чтобы дизайн выглядел премиально.

#### Acceptance Criteria

1. THE ProjectCard SHALL иметь полупрозрачный темный фон с альфа-каналом
2. THE ProjectCard SHALL применять backdrop-filter: blur(10px) для эффекта размытия
3. THE ProjectCard SHALL иметь тонкую светящуюся рамку
4. WHEN пользователь просматривает список проектов, THEN все карточки SHALL отображаться с glassmorphism эффектом
5. WHEN фон за карточкой изменяется, THEN эффект размытия SHALL быть визуально заметен

### Requirement 4: Визуализация Метрик

**User Story:** Как бизнес-заказчик, я хочу видеть визуальные индикаторы качества проектов, чтобы быстро оценивать их эффективность.

#### Acceptance Criteria

1. WHEN ProjectCard отображает метрику Faithfulness, THEN System SHALL показывать визуальный индикатор (прогресс-бар или цветной бейдж)
2. WHEN ProjectCard отображает метрику Precision, THEN System SHALL показывать визуальный индикатор
3. WHEN значение метрики высокое (>0.8), THEN индикатор SHALL быть зеленым
4. WHEN значение метрики среднее (0.5-0.8), THEN индикатор SHALL быть желтым
5. WHEN значение метрики низкое (<0.5), THEN индикатор SHALL быть красным
6. THE System SHALL отображать числовое значение метрики рядом с визуальным индикатором

### Requirement 5: Иконки Архитектуры

**User Story:** Как бизнес-заказчик, я хочу видеть визуальные символы типов архитектур, чтобы быстро понимать технологический стек проекта.

#### Acceptance Criteria

1. THE ProjectCard SHALL содержать область для SVG-иконки в верхней части карточки
2. WHEN тип архитектуры определен, THEN System SHALL отображать соответствующую иконку
3. WHEN иконка для типа архитектуры отсутствует, THEN System SHALL отображать placeholder иконку
4. THE System SHALL использовать абстрактные SVG-иконки для визуализации типов архитектур

### Requirement 6: Интерактивные Hover Эффекты

**User Story:** Как пользователь, я хочу видеть отклик интерфейса на мои действия, чтобы взаимодействие было приятным и современным.

#### Acceptance Criteria

1. WHEN пользователь наводит курсор на ProjectCard, THEN карточка SHALL плавно подниматься (transform: translateY(-5px))
2. WHEN пользователь наводит курсор на ProjectCard, THEN свечение границы SHALL усиливаться
3. THE System SHALL применять плавную анимацию (transition) для всех hover эффектов
4. WHEN пользователь убирает курсор с ProjectCard, THEN карточка SHALL плавно возвращаться в исходное положение
5. THE System SHALL использовать transition длительностью не менее 0.3s для плавности

### Requirement 7: Сохранение Функциональности

**User Story:** Как пользователь, я хочу, чтобы все существующие функции работали после редизайна, чтобы не потерять доступ к контенту.

#### Acceptance Criteria

1. WHEN редизайн применен, THEN все ссылки и навигация SHALL продолжать работать
2. WHEN редизайн применен, THEN все данные проектов SHALL отображаться корректно
3. WHEN редизайн применен, THEN страница Лаборатории SHALL сохранять свою функциональность
4. THE System SHALL сохранять все существующие компоненты и их логику
5. THE System SHALL изменять ТОЛЬКО стили (CSS), не затрагивая JavaScript логику

### Requirement 8: Мобильная Адаптивность и Производительность

**User Story:** Как пользователь мобильного устройства, я хочу, чтобы сайт работал плавно и текст был читаемым, чтобы комфортно просматривать проекты на телефоне.

#### Acceptance Criteria

1. WHEN пользователь открывает сайт на мобильном устройстве (ширина <= 768px), THEN backdrop-filter SHALL быть отключен для улучшения производительности
2. WHEN пользователь открывает сайт на мобильном устройстве, THEN фон карточек SHALL быть более непрозрачным (alpha >= 0.95) для читаемости текста
3. WHEN пользователь просматривает карточки на мобильном, THEN отступы между карточками SHALL быть не менее 1.5rem для предотвращения слияния
4. WHEN пользователь читает текст на мобильном, THEN line-height SHALL быть увеличен до 1.7 для лучшей читаемости
5. WHEN пользователь взаимодействует с карточками на мобильном, THEN hover эффекты SHALL быть упрощены (меньше transform, легче box-shadow)
6. WHEN устройство имеет настройку prefers-reduced-motion, THEN backdrop-filter и сложные анимации SHALL быть отключены
7. THE System SHALL обеспечивать 60fps на мобильных устройствах 2019+ года
